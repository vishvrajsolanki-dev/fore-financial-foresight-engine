# FORE — The Financial Foresight Engine

Cursor Hackathon Ahmedabad · "AI for Everyday Experiences"

**Product spec:** [docs/FORE_Blueprint.md](docs/FORE_Blueprint.md) is the canonical blueprint. An earlier FORE Blueprint 2 spec proposed JWT auth and PostgreSQL — those were deliberately **not** built; see [docs/FORE_Blueprint_2_SUPERSEDED.md](docs/FORE_Blueprint_2_SUPERSEDED.md) and [docs/ROADMAP.md](docs/ROADMAP.md) Tier 3.

One platform, three linked faces on a shared data spine:

- **PAST** — spending archetype (Euclidean distance to 5 fixed centroids) + burn-rate trend + zero-balance projection
- **DECIDE** — "can I afford X" chat, grounded by a real tool-call function (`canIAfford()`), never a guessed number
- **AHEAD** — single-session goal pace calculator + static peer-benchmark percentile panel

**The one thing judges should remember:** the model calling a real function live on stage instead of inventing the affordability math.

## Stack

```
Frontend/BE  : Next.js 14 App Router, TypeScript, Tailwind CSS + shadcn/ui, Recharts — on Vercel
ML service   : Python (FastAPI) — on Render, always-on instance (no cold starts on the demo path)
Chat/LLM     : Groq API + Llama 3.1, real tool-calling
State        : In-memory financial_context object — no database, no auth (see docs/ for why)
```

## Running locally

Two processes: the Python ML service (Render target) and the Next.js app (Vercel target).

```bash
# 1. ML service (FastAPI) — terminal 1
cd ml-service
python3 -m venv .venv && ./.venv/bin/pip install -r requirements.txt
./.venv/bin/uvicorn main:app --host 127.0.0.1 --port 8000

# 2. Next.js app — terminal 2 (points at the ML service via RENDER_ML_BASE_URL)
npm install
RENDER_ML_BASE_URL=http://127.0.0.1:8000 npm run dev
# open http://localhost:3000 → select a demo persona
```

Copy `.env.example` to `.env.local` and set `GROQ_API_KEY` to enable real Llama tool-calling on
DECIDE. Without a key, DECIDE uses a deterministic fallback that **still calls the real
`canIAfford()` function**, so the affordability numbers remain honest.

Regenerate the synthetic data (rarely needed — it's checked in):

```bash
python3 scripts/generate_data.py
npm run verify
```

Run the FORE_Blueprint §8 demo spine smoke test (requires ML service on :8000 and Next.js on :3000):

```bash
npm run smoke
```

## What's implemented (Tier 1 core)

- **ML service** (`ml-service/`): `/classify` (Euclidean distance to 5 locked centroids),
  `/burn-rate` (running-balance trend + runway projection), `/can-i-afford` (reuses the regressor
  with a hypothetical expense inserted). CORS scoped to the Vercel/localhost origin, not `*`.
- **Data** (`data/`): 5 distinct synthetic personas (~120 txns each, all 5 archetypes) + full benchmark JSON
  (5 income brackets × 3 city tiers), generated by `scripts/generate_data.py`.
- **PAST**: archetype label + radar chart + burn-rate stats + running-balance line.
- **DECIDE**: chat wired to Groq tool-calling; the model must call `canIAfford()` before stating
  any number. UI shows a "checked your numbers" badge and a 2s graceful-wait state.
- **AHEAD**: goal-pace calculator + peer-benchmark percentiles; reflects the latest DECIDE verdict
  via the shared `financial_context`.

## Deployment
